waveform-playlist (React) — latency compensation demo

Recording through headphones and a mic always lands late — delayed by the device's round-trip audio latency.

  • You'll hear it as a "flam" against the guide track.
  • You'll see it as a waveform offset.

This demo proves the fix against the React layer of waveform-playlist (@waveform-playlist/browser + @waveform-playlist/recording), an AudioWorklet-based multitrack editor: measure round-trip latency with the <latency-test> component, then apply that value to a recording's start offset.

Like the dawcore demo before it, this library already applies its own internal compensation to every recording (based on AudioContext.outputLatency + the scheduler's lookahead — both browser/library estimates, not anything externally measured). So the "uncalibrated" take below isn't truly uncompensated — it's the library's own built-in guess. The "calibrated" take replaces that guess with the value <latency-test> actually measured. The alignment proof in step 7 shows both numbers side by side, so the real comparison stays honest.
